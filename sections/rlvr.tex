
\section*{Extra Credit 5: Reinforcement Learning with Verifiable Rewards (RLVR)}

\subsection*{Methodology}
In this experiment, we evaluated the use of Group Relative Policy Optimization (GRPO), a reinforcement learning framework, to teach the model alien arithmetic. Unlike Supervised Fine-Tuning (LoRA) where the model is explicitly trained on ground-truth reasoning trajectories, RLVR relies on a verifiable reward signal. The model generated responses to the alien math problems, and we provided a binary reward ($1.0$ for an exact match of the final integer, $0.0$ otherwise) using the \texttt{exact\_match\_reward} function. The model was given a basic prompt without any system definitions of the operators.

\subsection*{Analysis and Discussion}

The RLVR approach failed to generalize, achieving only an \textbf{8.0\% overall accuracy}. This represents a severe degradation compared to our prior supervised LoRA approach. This discrepancy highlights the fundamental difference between dense supervision and sparse rewards.

In Supervised Fine-Tuning (LoRA), the model is trained using \textbf{dense supervision} (teacher forcing). It is explicitly provided with the exact correct target string (e.g., \texttt{Final Answer: 16}) and receives immediate token-by-token cross-entropy feedback, directly penalizing it for generating anything else. This direct guidance allows the model to quickly map the alien \texttt{flarn} token to its pre-trained concept of addition.

Conversely, RLVR suffers from the \textbf{sparse reward problem}. Under GRPO, the model generates an entire response without seeing the target. If it is wrong, it receives a $0.0$ reward, which tells it the answer was incorrect but provides no information about what the right answer should have been. Because we did not provide a system prompt defining the alien operators, the model was forced to rely purely on random exploration. To learn that \texttt{flarn(9, 7)} means $9 + 7 = 16$, the model would have to randomly hallucinate the exact number 16. The probability of randomly generating the correct multi-digit integer by pure chance is infinitesimally small. As a result, the model almost never received a positive reward ($1.0$) during the early stages of training, causing the policy gradient to flatline.

\subsection*{Qualitative Examples}

The model's failure modes reveal that instead of learning the alien operators, it fell back on its pre-trained arithmetic priors, commonly defaulting to multiplication:
\begin{itemize}
    \item \textbf{Problem:} \texttt{glorp(flarn(9, 7))} $\rightarrow$ Expected: 16 (Addition)
    \item \textbf{Model Output:} \texttt{Final Answer: 63} (Multiplication)
    
    \item \textbf{Problem:} \texttt{glorp(snib(13, 10))} $\rightarrow$ Expected: 3 (Subtraction)
    \item \textbf{Model Output:} \texttt{Final Answer: 130} (Multiplication)
\end{itemize}

Because the model lacked the guidance of supervised reasoning trajectories and failed to randomly stumble upon enough positive rewards to infer the rules, GRPO proved entirely ineffective for adapting the model to zero-shot alien syntax in this restricted training window.
