\section*{5. Reinforcement Learning with Verifiable Rewards (RLVR)}

\subsection*{Methodology}
We trained the model on alien arithmetic using Group Relative Policy Optimization (GRPO). Instead of explicit reasoning trajectories (as in LoRA), the model received a sparse binary reward ($1.0$ for an exact integer match, $0.0$ otherwise) without any system definitions of the alien operators.

RLVR catastrophically failed compared to supervised LoRA due to the \textbf{sparse reward problem}. LoRA utilizes dense supervision (teacher forcing), providing immediate token-level gradients that quickly map alien operators to pre-trained concepts. GRPO relies entirely on random exploration. Without a system prompt defining the operators, the model had to randomly hallucinate the exact correct multi-digit answer to receive a positive reward. The near-zero probability of this occurring caused the policy gradient to flatline.

\subsection*{Qualitative Examples}
Lacking explicit guidance, the model defaulted to its pre-trained arithmetic priors (typically multiplication):
\begin{itemize}
    \item \textbf{Problem:} \texttt{glorp(flarn(9, 7))} $\rightarrow$ Expected: 16 (Addition) | \textbf{Model Output:} 63
    \item \textbf{Problem:} \texttt{glorp(snib(13, 10))} $\rightarrow$ Expected: 3 (Subtraction) | \textbf{Model Output:} 130
\end{itemize}
